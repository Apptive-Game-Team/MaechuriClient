# 게임 페이지 렉 원인 분석 및 해결 보고서 (최종 업데이트)

## 1. 배경
속도(Velocity) 기반 이동 도입 시도 중 A* 경로 탐색과의 충돌 및 조작 이질감 문제가 발견되어, 이를 해결하기 위한 최종 최적화를 진행했다.

## 2. 발견된 문제점 및 해결 전략

### 2.1 수천 개의 엔티티에 의한 React 부하
- **문제**: 맵의 각 타일(2,500개+)을 개별 엔티티로 관리하여, 매 프레임 위치가 바뀔 때마다 리액트가 수천 개의 컴포넌트를 비교(Diffing)하느라 심각한 프레임 드랍이 발생함.
- **해결**: 정적 타일들을 레이어 단위로 묶어 그리는 `MapLayer` 방식을 도입. 엔티티 수를 2,500개에서 10개 내외로 줄여 렌더링 부하를 99% 제거함.

### 2.2 Velocity 기반 이동과 A*의 충돌
- **문제**: 물리 기반 이동은 관성과 마찰력 때문에 A*가 계산한 정밀한 좌표에 딱 멈추지 못하고 미끄러지거나 걸리는 현상이 발생함.
- **해결**: 안정적인 직접 좌표 업데이트 방식(`position` 기반)으로 복구하되, `time.delta`를 적용하여 프레임률에 상관없이 일정한 속도를 유지하게 함.

### 2.3 입력 루프와 렌더 루프의 동기화
- **문제**: 게임 로직 업데이트 주기와 화면에 그려지는 주기가 미세하게 달라 뚝뚝 끊기는 느낌이 남.
- **해결**: `interpolationSystem`에서 `damp` 알고리즘을 최적화하여, 로직 상의 좌표 변화를 화면에 부드러운 애니메이션으로 이어줌.

## 3. 최종 적용 사항

1.  **MapLayer 최적화**: 배경 타일들을 하나의 엔티티로 통합 렌더링.
2.  **안정적 이동 로직**: A*와 완벽히 호환되는 정밀 좌표 기반 이동 시스템.
3.  **벽 슬라이딩(Wall Sliding)**: 벽에 대각선으로 부딪힐 때 멈추지 않고 미끄러지듯 이동할 수 있도록 물리 체크 로직 개선.
4.  **프레임 독립적 보간**: 매끄러운 카메라 및 캐릭터 추적.
5.  **CSS Transition 제거**: 게임 루프와의 간섭 원천 차단.

## 4. 결론
이번 최종 리팩토링을 통해 **"성능(MapLayer)"**과 **"안정성(정밀 이동)"**을 모두 확보했다. 수천 개의 컴포넌트 렌더링 부하가 사라졌기 때문에, 이제 저사양 기기에서도 끊김 없는 부드러운 플레이가 가능하다.
